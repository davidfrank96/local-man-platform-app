## Title
The Local Man — Agent Rules

1. Read the docs before making any code change.
2. Do not invent features outside the PRD.
3. Do not change architecture without updating DECISIONS.md.
4. Keep code typed and maintainable.
5. Prefer simple patterns over clever patterns.
6. Keep public and admin concerns cleanly separated.
7. Validate all write operations.
8. Update docs after meaningful changes.
9. Add tests for critical logic.
10. If unsure, choose the smallest safe implementation.
11. Avoid placeholder junk and fake implementations.
12. Break large tasks into smaller verifiable tasks.
